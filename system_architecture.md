# System Architecture Document

**Project:** InsureRoute  
**Domain:** Predictive AI, Algorithmic Routing, InsurTech  

---

## 1. High-Level Architecture Overview

InsureRoute is engineered on a strictly decoupled, microservices-inspired paradigm. The application separates the heavy mathematical processing capabilities—machine learning anomaly detection, graph routing, and dynamic pricing calculations—from the client-side visualization loop.

This physical separation ensures:
1.  **Computational Stability:** Massive matrix operations inside the Scikit-Learn pipeline do not block the React Render thread.
2.  **Scalability:** Each tier can be hosted and scaled independently (e.g., deploying the frontend to a CDN like Vercel, and the backend to an AWS EC2 instance optimized for tabular processing).
3.  **Security:** All proprietary actuarial formulas remain secure on the server, exposing only calculated outputs to the user browser.

## 2. Infrastructure Tiers

### 2.1. The Client Tier (Frontend)
The user-facing application is built for extreme rendering efficiency and state management.

*   **Framework:** React 18 & Vite
*   **Design Tokens:** Tailwind CSS (configured via `tailwind.config.js` for isolated dark-mode telemetry aesthetics).
*   **State Management:** Asynchronous Fetching hooks (`useEffect` polling structures executing against REST endpoints).
*   **Role:** Acts as the "Digital Twin." It parses the complex JSON blobs generated by the backend and orchestrates the visual state of the Dashboard, KPI trackers, interactive map interfaces, and the chronological Logs panel.

### 2.2. The Intermediate API Layer (Middleware)
Provides the standardized communication protocol bridging the UI and the Data Pipeline.

*   **Framework:** FastAPI (Python)
*   **Data Validation:** Pydantic models automatically sanitize and structure incoming payloads (preventing type-mismatch routing crashes).
*   **Role:** Exposes endpoints (e.g., `GET /data`, `POST /inject-disruption`). It orchestrates the sequence of events globally—from fetching the data snapshot, calling the ML model, requesting the route, and triggering the pricing engine based on the previous outputs.

### 2.3. The Computational Tier (Backend Engines)
The core mathematical execution environment is subdivided into three distinct parallel modules.

#### A. Predictability Engine (`model.py`)
*   **Algorithms:** `sklearn.ensemble.IsolationForest`
*   **Function:** Ingests live telemetry parameters (cargo type, monsoon indicators, geographic timestamps). Processes these vectors against historical scaled subsets. Outputs a strict `anomaly_score` mapping to risk.

#### B. Algorithmic Routing Engine (`graph_engine.py`)
*   **Algorithms:** `NetworkX`, `Dijkstra’s Algorithm`
*   **Function:** Loads an explicit coordinate-mapped topological layout. If the Predictability Engine detects an anomaly `< -0.15`, the engine instantly targets the active vector edge, synthetically bloats the path weight, and calculates a new optimized traversal array (avoiding the failure zone).

#### C. Actuarial Pricing Engine (`pricing_engine.py`)
*   **Function:** Converts arbitrary risk likelihood into physical fiat capital representation. It calculates base premiums against cargo scalar values and applies categorical disruption multipliers (Weather mult, Perishable mult) to deliver both unmitigated liability estimates and newly hedged risk values.

## 3. Request Lifecycle Diagram (Data Flow)

The system operates around a continuous polling loop or a manual trigger (Disruption Injection).

1.  **Transmission:** The React UI dispaches an HTTP Request to the API with payload constraints.
    ```json
    { "cargo_value": 70000, "perishable": true, "monsoon": true }
    ```
2.  **Ingestion:** `/backend/api.py` intercepts the request, maps it via Pydantic, and calls the `run_tick()` orchestrator.
3.  **Inference Allocation:** A generated or live network state is cast to `/backend/model.py`. Historical baselines are drawn. Anomaly scoring bounds are executed.
4.  **Topology Execution:**
    *   *If Anomaly > Threshold:* The baseline shortest path is requested from `/backend/graph_engine.py`.
    *   *If Anomaly < Threshold:* `/backend/graph_engine.py` is commanded to instantiate a shadow-graph copy, penalize the problematic vertex connection, and calculate the bypass.
5.  **Financial Mapping:** `/backend/pricing_engine.py` executes actuarial arithmetic contrasting the original risk index against the bypass risk index, deducing `savings_pct`.
6.  **Serialization:** `api.py` consolidates all module outputs into a singular complex JSON hierarchy.
7.  **Client Resolution:** The React Dashboard parses the JSON context, updating SVG graph lines, recoloring KPI elements to danger states (red), and logging the telemetry trace verbatim to the user UI.

## 4. Sub-system Configurations

### Data Formatting (`preprocessing.py` & `data_loader.py`)
The pipeline is designed to eliminate nulls and standardise non-uniform datasets before ML Inference.
*   Converts boolean values to integer matrices.
*   Executes `StandardScaler` on vast numeric anomalies (temperatures, traffic throughput) to ensure model trees partition logically across multiple axes.

### CORS & Security Implementation
The FastAPI server implements strict Cross-Origin Resource Sharing (CORS) middleware to prevent unauthorized API interactions while allowing Vite's internal development ports (`localhost:3000`, `localhost:5173`) to proxy requests directly to the core orchestrator without pre-flight blockages.
